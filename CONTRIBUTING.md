# Contributing

This repository hosts an article proposing the VOUCH Framework for managing cognitive debt. It's an open problem space, not a finished standard.

## How to participate

**Challenge the framework.** If you think cognitive debt is real but VOUCH is the wrong approach, open an issue explaining why. The best contributions are the ones that break the model and force us to think harder.

**Propose alternatives.** If git notes are the wrong storage mechanism, if AST hashing is too coarse (or too fine), if the endorsement model has blind spots — write it up. Issues and pull requests are both welcome.

**Share your data.** If you've measured anything resembling cognitive debt in your own organization — review times, incident correlation with AI-generated code, onboarding friction — we want to hear about it. Anecdotes are valuable. Data is better.

**Build tooling.** The `vouch` CLI described in the article is a starting point. If you build a prototype (in any language, with any storage backend), link it here. The framework is language-agnostic and implementation-agnostic by design.

## What this is not

- Not a spec. Not an RFC. Not a standard.
- Not affiliated with any company or organization.
- Not prescriptive — if your team's approach to cognitive debt looks nothing like VOUCH, that's fine. The goal is to start measuring, not to converge on a single tool.

## Ground rules

- Argue ideas, not people.
- Data and references over opinions.
- If you disagree with a claim in the article and have evidence, open an issue. We'll update the article or add a counterpoint.
